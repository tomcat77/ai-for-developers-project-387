# Описание приложения Calls Calendar

## Общее описание
В проекте есть две роли: владелец календаря и гости.

В этом проекте нет регистрации и авторизации. Владелец календаря — один заранее заданный профиль. Гость бронирует слоты без создания аккаунта и без входа в систему.

Владелец календаря может:

1. Создавать типы событий. Для каждого типа события задает id, название, описание и длительность в минутах.
1. Просматривает страницу предстоящих встреч, где в одном списке показаны бронирования всех типов событий.

Гость:

1. Может посмотреть страницу с видами брони, где доступно название, описание и длительность.
1. Выбирает тип события, открывает календарь и выбирает свободный слот.
1. Создает бронирование на выбранный слот.

## Правило занятости
На одно и то же время нельзя создать две записи, даже если это разные типы событий.

Внутреннюю реализацию, стек и структуру проекта вы выбираете сами. На этом этапе мы договариваемся только о внешнем поведении системы: какие сценарии поддерживаются, какие данные передаются и какие ограничения действуют при бронировании.

## Описание сущностей

### EventType (Тип события)
- `id` — уникальный идентификатор (string вида `DEFAULT` или `QUICK`)
- `name` — название (string), например, "Созвон 15 мин"
- `description` — описание (string), например, "Быстрый созвон на 15 мин. с четкой повесткой"
- `durationMinutes` — длительность в минутах (integer), например: 15

### Booking (Бронирование)
- `id` — уникальный идентификатор (uuid, генерится автоматически)
- `eventTypeId` — ссылка на тип события (string)
- `startTime` — дата и время начала
- `guestName` — имя гостя, например, "Константин"
- `guestContact` — контакт гостя, например, `mail@server.com`

### CalendarOwner (Владелец календаря)
- `id` — уникальный идентификатор (uuid, генерится автоматически)
- `name` — имя, например, "Вольдемар"
- `contact` — контакт для связи, например, `voldemar@hogwarts.uk`
