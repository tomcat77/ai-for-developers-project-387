# Сценарий тестирования: Создание нового типа встреч

## Описание сценария
Владелец календаря создает новый тип события с заданными параметрами (id, название, описание, длительность).

## Предусловия
- Приложение запущено и доступно
- Владелец календаря имеет доступ к системе

## Шаги выполнения

### Шаг 1: Открыть страницу управления типами событий
**Действие:** Владелец календаря открывает страницу с типами событий.
**Ожидаемый результат:** Отображается список существующих типов событий (если есть) и кнопка для создания нового типа.

### Шаг 2: Нажать кнопку создания типа события
**Действие:** Владелец нажимает на кнопку создания нового типа события.
**Ожидаемый результат:** Открывается диалог с заголовком **"Создать тип события"** и формой с полями:
- **Название** (placeholder: "Например: Консультация")
- **Описание**
- **Длительность (минуты)**
- **Цвет**
- Кнопки: **"Отмена"**, **"Сохранить"** (неактивна при невалидной форме)

### Шаг 3: Заполнить поле "Название"
**Действие:** Ввести название типа события в поле "Название", например "Консультация".
**Ожидаемый результат:** Поле заполнено значением "Консультация". При попытке сохранить пустое поле отображается ошибка **"Название обязательно"**, при вводе менее 2 символов — **"Минимум 2 символа"**.

### Шаг 4: Заполнить поле "Описание"
**Действие:** Ввести описание типа события в поле "Описание", например "Детальная консультация по проекту".
**Ожидаемый результат:** Поле заполнено значением "Детальная консультация по проекту". При попытке сохранить пустое поле отображается ошибка **"Описание обязательно"**.

### Шаг 5: Заполнить поле "Длительность (минуты)"
**Действие:** Ввести длительность в поле "Длительность (минуты)", например 30.
**Ожидаемый результат:** Поле заполнено значением 30. Ошибки валидации:
- При пустом поле: **"Длительность обязательна"**
- При значении < 1: **"Минимум 1 минута"**
- При значении > 480: **"Максимум 480 минут"**

### Шаг 6: Выбрать цвет (опционально)
**Действие:** При необходимости выбрать цвет для типа события через color picker.
**Ожидаемый результат:** Выбран цвет (по умолчанию #ff9800).

### Шаг 7: Нажать кнопку "Сохранить"
**Действие:** Владелец нажимает кнопку **"Сохранить"**.
**Ожидаемый результат:** 
- Диалог закрывается
- В нижней части экрана отображается уведомление (snackbar): **"Тип события создан"**
- Новый тип события отображается в списке типов

## Постусловия
- Новый тип события "CONSULTATION" доступен в системе
- Гости могут видеть этот тип события при бронировании
- Тип события можно выбрать при создании бронирования

## Альтернативные сценарии

### A1: Попытка создать тип с ошибкой сервера
**Действие:** При попытке сохранить тип и получении ошибки от сервера.
**Ожидаемый результат:** Отображается уведомление: **"Не удалось создать тип события"**

### A2: Незаполненные обязательные поля
**Действие:** Попытка сохранить тип события с незаполненными обязательными полями (Название, Описание, Длительность).
**Ожидаемый результат:** 
- Кнопка "Сохранить" неактивна при невалидной форме
- Отображаются ошибки валидации:
  - Для Названия: **"Название обязательно"** или **"Минимум 2 символа"**
  - Для Описания: **"Описание обязательно"**
  - Для Длительности: **"Длительность обязательна"**
