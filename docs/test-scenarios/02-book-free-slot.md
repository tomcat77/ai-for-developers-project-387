# Сценарий тестирования: Успешное бронирование свободного слота

## Описание сценария
Гость просматривает доступные типы событий, выбирает подходящий тип, открывает календарь, выбирает свободный слот и успешно создает бронирование.

## Предусловия
- Приложение запущено и доступно
- В системе создан хотя бы один тип события (например, "Созвон 15 мин")
- Для выбранного типа события есть свободные слоты в календаре
- Гость не авторизован в системе (доступ без регистрации)

## Шаги выполнения

### Шаг 1: Открыть страницу с видами брони
**Действие:** Гость открывает страницу с доступными типами событий.
**Ожидаемый результат:** Отображается список доступных типов событий с названием, описанием и длительностью каждого типа.

### Шаг 2: Выбрать тип события
**Действие:** Гость выбирает интересующий тип события из списка, например "Созвон 15 мин".
**Ожидаемый результат:** 
- Выбранный тип события подсвечен/активен
- Отображается кнопка или ссылка для перехода к выбору слота

### Шаг 3: Перейти к выбору слота
**Действие:** Гость нажимает на выбранный тип события для перехода к выбору слота.
**Ожидаемый результат:** Открывается страница выбора даты и времени с календарем. Отображаются свободные временные интервалы для выбранного типа события.

### Шаг 4: Выбрать дату в календаре
**Действие:** Гость выбирает дату в календаре, на которую хочет забронировать встречу.
**Ожидаемый результат:** 
- Выбранная дата отображается активной
- Под календарем или в боковой панели отображаются доступные временные слоты для этой даты

### Шаг 5: Выбрать свободный временной слот
**Действие:** Гость выбирает свободный временной слот из списка доступных (например, "10:00 - 10:15").
**Ожидаемый результат:** 
- Выбранный слот подсвечен/активен
- Открывается страница "Подтверждение" с кнопками **"Назад"**, **"Пропустить"**, **"Забронировать"**
- Отображаются детали бронирования:
  - **Событие:** [название типа]
  - **Дата:** [выбранная дата]
  - **Время:** [выбранное время]
  - **Длительность:** [N] мин

### Шаг 6: Указать контактные данные (опционально)
**Действие:** Гость активирует чекбокс **"Указать контактные данные"** и заполняет поля:
- **Имя:** например, "Александр" (обязательно при включенном чекбоксе)
- **Email:** например, "alex@example.com" 
- **Телефон:** опционально
- **Заметки:** опционально, максимум 500 символов

**Ожидаемый результат:** Поля заполнены. При пустом поле "Имя" отображается ошибка **"Введите имя"**. При некорректном email отображается ошибка **"Введите корректный email"**.

### Шаг 7: Пропустить ввод контактов (альтернатива)
**Действие:** Гость нажимает кнопку **"Пропустить"**, не указывая контактные данные.
**Ожидаемый результат:** Бронирование создается без контактной информации гостя.

### Шаг 8: Подтвердить бронирование
**Действие:** Гость нажимает кнопку **"Забронировать"**.
**Ожидаемый результат:** 
- Отображается уведомление (snackbar): **"Бронирование успешно создано!"** (отображается 5 секунд)
- Появляется подтверждение с деталями бронирования (тип события, дата, время, длительность)
- ID бронирования отображается для справки
- Выбранный слот больше не отображается как свободный в календаре

### Шаг 9: Проверить страницу предстоящих встреч (опционально)
**Действие:** Владелец календаря открывает страницу предстоящих встреч.
**Ожидаемый результат:** Созданное бронирование отображается в списке предстоящих встреч с данными гостя (имя, контакт) и деталями встречи.

## Постусловия
- Бронирование создано в системе с уникальным ID
- Выбранный временной слот помечен как занятый
- Владелец календаря видит новое бронирование в списке предстоящих встреч
- Гость получил подтверждение бронирования

## Альтернативные сценарии

### A1: Незаполненное поле "Имя" при активном чекбоксе контактов
**Действие:** Чекбокс **"Указать контактные данные"** активирован, но поле **"Имя"** не заполнено.
**Ожидаемый результат:** 
- Кнопка **"Забронировать"** неактивна при невалидной форме
- Отображается ошибка валидации: **"Введите имя"**
